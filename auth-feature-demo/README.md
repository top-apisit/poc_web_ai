# Authentication Feature - Demo Project

Simple username/password authentication for web application demo.

## 📁 Folder Structure

```
auth-feature-demo/
├── 01-ba/           # Business Analyst - Requirements
├── 02-sa/           # Solution Architect - Architecture
├── 03-ux-ui/        # UX/UI Designer - Design Specs
├── 04-be/           # Backend Engineer - API Implementation
├── 05-fe/           # Frontend Engineer - UI Implementation (YOUR FOCUS)
├── 06-qa/           # QA - Test Cases
└── 07-devops/       # DevOps - Deployment Config
```

## 🎯 Feature Scope

### In Scope (MVP)
- [x] User registration (username, email, password)
- [x] User login (username/email + password)
- [x] User logout
- [x] Session management (JWT)
- [x] Basic form validation

### Out of Scope (Future)
- Social login (Google, GitHub)
- Two-factor authentication
- Password reset via email
- Remember me functionality
- Account verification email

## 🛠 Tech Stack

| Layer | Technology |
|-------|------------|
| Frontend | React + TypeScript + Tailwind |
| Backend | Node.js + Express/Hono |
| Database | PostgreSQL |
| Auth | JWT (JSON Web Tokens) |

## 🚀 Quick Start

1. Review BA requirements in `01-ba/`
2. Understand architecture in `02-sa/`
3. Check UI design specs in `03-ux-ui/`
4. Backend API docs in `04-be/`
5. **Implement frontend** in `05-fe/` (your task)
6. Run QA tests from `06-qa/`
7. Deploy using `07-devops/`
